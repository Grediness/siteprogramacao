HTML

O front-end é a parte de um site ou aplicativo que o usuário interage diretamente. É a interface visual, tudo aquilo que você vê e com o que você interage em uma página da web. Pense nos botões, menus, formulários, imagens e animações: tudo isso faz parte do front-end.

O HTML: é a estrutura da página. Ele define o conteúdo e a organização dos elementos, como títulos, parágrafos, imagens e vídeos.

Em qualquer página web que você acessa, por trás da interface visual que você vê, existe uma estrutura fundamental construída com HTML (HyperText Markup Language). As tags de estrutura básicas são os pilares dessa fundação, os elementos essenciais que definem a espinha dorsal de todo documento HTML. Elas informam ao navegador como interpretar e organizar o conteúdo da página, estabelecendo os limites e o propósito de cada seção principal. Sem essas tags, o navegador não saberia onde começa e termina o documento, qual é o cabeçalho que define as informações sobre a página e qual é o corpo que contém o conteúdo visível ao usuário. Dominar essas tags é o primeiro passo crucial para qualquer pessoa que deseja criar e entender a estrutura da web.

html: Define a raiz de um documento HTML. Todos os outros elementos devem ser descendentes deste elemento.

head: Contém metadados sobre o documento HTML, como o título do documento, conjunto de caracteres, estilos CSS, scripts JavaScript e outras informações que não são exibidas diretamente na página

title: Define o título do documento que é exibido na barra de título do navegador ou na aba da página.

body: Contém o conteúdo visível do documento HTML, como texto, imagens, links, etc.

Uma vez estabelecida a estrutura fundamental de um documento HTML com as tags básicas, o próximo passo crucial é preencher essa estrutura com o conteúdo propriamente dito. É aqui que entram em cena as Tags de Conteúdo. Essas tags são responsáveis por dar significado e forma ao texto, às imagens, às listas e a outros elementos que compõem a parte visível de uma página web. Elas permitem organizar o texto em parágrafos distintos, definir títulos e subtítulos com diferentes níveis de importância, criar listas ordenadas e não ordenadas, inserir imagens que enriquecem a informação e até mesmo criar links que conectam diferentes partes da web. Em essência, as Tags de Conteúdo são as ferramentas que transformam um esqueleto HTML em uma página rica e informativa para o usuário.

h1 a h6: Define títulos de diferentes níveis de importância (h1 é o mais importante).

p: Define um parágrafo de texto.

a: Define um hiperlink para outras páginas ou locais dentro da mesma página. O atributo href especifica o destino do link.

img: Incorpora uma imagem no documento. O atributo src especifica o caminho para a imagem, e alt fornece um texto alternativo para a imagem.

ul: Define uma lista não ordenada (com marcadores).

ol: Define uma lista ordenada (com números).

li: Define um item de lista dentro de uma lista não ordenada (ul) ou ordenada (ol).

span: Um container embutido usado para marcar partes de texto para aplicar estilos CSS ou manipular com JavaScript. Não possui significado semântico próprio.

div: Um container de bloco usado para agrupar outros elementos HTML. É frequentemente usado com CSS para layout e estilo. Não possui significado semântico próprio.

br: Insere uma quebra de linha.

hr: Representa uma quebra temática entre elementos de nível de parágrafo (uma linha horizontal).

pre: Define texto pré-formatado. O texto dentro deste elemento é exibido em uma fonte de largura fixa e preserva espaços e quebras de linha.

blockquote: Indica uma seção que é citada de outra fonte. Os navegadores geralmente adicionam recuo a esses elementos.

Após definirmos a estrutura básica e populá-la com o conteúdo visível através das tags apropriadas, chegamos a um nível mais profundo e significativo da organização do HTML: as Tags de Semântica. Enquanto as tags de estrutura fornecem o esqueleto e as tags de conteúdo dão forma ao que vemos, as tags semânticas enriquecem o HTML com significado. Elas comunicam ao navegador e a outras tecnologias (como mecanismos de busca e leitores de tela) o propósito do conteúdo dentro de cada seção. Em vez de simplesmente usar div s para agrupar blocos de conteúdo, as tags semânticas como article, nav, aside, header e footer descrevem o papel daquele conteúdo na estrutura geral do documento. Essa clareza semântica melhora a acessibilidade, otimiza o SEO (Search Engine Optimization) e torna o código mais compreensível e manutenível, elevando a qualidade e a inteligência da sua página web.

header: Representa um conteúdo introdutório ou um conjunto de links de navegação para sua seção de conteúdo pai.

nav: Define um conjunto de links de navegação.

main: Especifica o conteúdo principal do documento. Deve haver apenas um elemento main por documento.

article: Representa uma composição autocontida em um documento, página, aplicativo ou site (por exemplo, uma postagem de blog, um artigo de jornal).

section: Agrupa conteúdo tematicamente relacionado.

aside: Representa conteúdo relacionado tangencialmente ao conteúdo ao redor (como barras laterais, pull quotes, etc.).

footer: Representa o rodapé de um documento ou seção, geralmente contendo informações sobre o autor, direitos autorais, links para termos de uso, informações de contato, etc

figure: Representa conteúdo autocontido, como ilustrações, diagramas, fotos, listagens de código, etc., que são referenciados no texto principal.

figcaption: Fornece uma legenda para um elemento figure.

mark: Representa texto marcado ou realçado para fins de referência ou notação.

time: Representa um período de tempo específico

Interagir com o usuário é um aspecto fundamental da web moderna, e as Tags de Formulário são os blocos de construção essenciais para essa comunicação. Elas fornecem os mecanismos pelos quais os usuários podem inserir dados, fazer escolhas e enviar informações para o servidor. Desde simples campos de texto para digitar um nome ou e-mail, até caixas de seleção para múltiplas opções, botões de rádio para escolhas únicas e áreas de texto maiores para mensagens, as tags de formulário oferecem uma variedade de controles interativos. A tag form atua como o container principal, delimitando a área onde esses elementos de entrada residem, enquanto tags como input, textarea, select e button definem os diferentes tipos de campos e ações que o usuário pode realizar. Essenciais para coleta de dados, pesquisas, logins e muitas outras funcionalidades interativas, as Tags de Formulário são a porta de entrada para a participação ativa do usuário em uma página web.

form: Define um formulário HTML usado para entrada do usuário.

input: Define um controle de entrada para dados do usuário (por exemplo, texto, botões, caixas de seleção). O atributo type especifica o tipo de controle.

textarea: Define um controle de entrada multiline (uma área de texto).

button: Define um botão clicável.

select: Define uma lista suspensa (dropdown).

option: Define uma opção em uma lista suspensa.

label: Define um rótulo para um elemento de entrada. Ajuda na acessibilidade.

label: Define um rótulo para um elemento de entrada. Ajuda na acessibilidade.

legend: Define uma legenda para um elemento fieldset.

Quando a apresentação organizada de dados em linhas e colunas se torna crucial, as Tags de Tabela entram em cena como a solução estrutural do HTML. Elas oferecem um conjunto de elementos interligados que permitem dispor informações de forma clara e concisa. A tag table atua como o contêiner principal, definindo os limites da tabela. Dentro dela, a tag tr demarca cada linha, enquanto as tags th são utilizadas para as células de cabeçalho (geralmente exibidas em negrito) e as tags td para as células de dados propriamente ditas. Elementos opcionais como caption para um título descritivo, thead, tbody e tfoot para agrupar semanticamente as seções da tabela, e colgroup e col para formatação de colunas, enriquecem ainda mais a capacidade de apresentar dados tabulares de maneira acessível e visualmente organizada. As Tags de Tabela são, portanto, ferramentas poderosas para exibir informações complexas de forma estruturada e fácil de entender na web.

table: Define uma tabela.

tr: Define uma linha em uma tabela.

th: Define uma célula de cabeçalho em uma tabela

Isso aqui seria a estrutura mais basica de um html

!DOCTYPE html

html lang="en"

head

meta charset="UTF-8"

meta name="viewport" content="width=device-width, initial-scale=1.0"

title>Document

/head

body

/body

/html

!DOCTYPE html: Esta declaração informa ao navegador que o documento é um documento HTML5. É a primeira coisa que aparece em um documento HTML e garante que o navegador renderize a página no modo padrão mais recente.

html lang="en": Esta é a tag raiz do documento HTML. Tudo o que está dentro dela é considerado parte da página web. O atributo lang="en" especifica o idioma principal do documento como inglês. Isso é útil para acessibilidade e para mecanismos de busca.

head: Esta seção contém informações sobre o documento HTML que não são exibidas diretamente na página. Ela inclui coisas como o título da página, links para arquivos CSS (para estilização), metadados (informações sobre o documento), e links para scripts.

meta charset="UTF-8": Esta tag define a codificação de caracteres para o documento como UTF-8. O UTF-8 é uma codificação de caracteres universal que suporta a maioria dos caracteres do mundo, garantindo que o texto seja exibido corretamente em diferentes sistemas e navegadores.

meta name="viewport" content="width=device-width, initial-scale=1.0": Esta tag de metadados configura a viewport para dispositivos móveis.

width=device-width: Define a largura da viewport para a largura do dispositivo.

initial-scale=1.0: Define o nível de zoom inicial quando a página é carregada pela primeira vez para 100%. Isso garante que a página seja renderizada corretamente em diferentes tamanhos de tela

titleDocument/title: Esta tag define o título do documento que aparece na aba do navegador ou na barra de título da janela. No seu exemplo, o título está como "Document", mas geralmente é algo mais descritivo sobre o conteúdo da página.

body: Esta seção contém todo o conteúdo visível da página web, como texto, imagens, links, vídeos, formulários e outros elementos interativos. No seu exemplo, o body está vazio, o que significa que a página renderizada seria em branco

Em resumo, este código HTML básico estrutura um documento web fundamental, definindo o tipo de documento, o idioma, informações de configuração importantes no head, e preparando o espaço no body para o conteúdo que será exibido ao usuário.
